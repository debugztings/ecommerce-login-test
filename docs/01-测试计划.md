# 电商网站登录模块测试计划

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 项目名称 | 电商网站登录模块测试 |
| 测试对象 | 电商网站用户登录功能（Web端） |
| 测试人员 | ztings |
| 测试时间 | 2026.06.25 - 2024.06.30 |

## 二、测试范围

### 功能范围
- 用户名/密码登录
- 记住密码功能
- 忘记密码入口
- 登录失败锁定机制
- 登录成功跳转

### 不包含范围
- 注册功能（属于注册模块）
- 第三方登录（属于第三方集成模块）
- 找回密码流程（属于密码管理模块）

## 三、测试环境

| 环境 | 地址 | 浏览器 |
|------|------|--------|
| 开发环境 | dev.example.com | Chrome 120+ |
| 测试环境 | test.example.com | Chrome 120+, Firefox 120+, Edge 120+ |
| 生产环境 | www.example.com | 同上 |

## 四、测试策略

### 4.1 功能测试
- 使用等价类划分、边界值分析法设计测试用例
- 覆盖正常登录、密码错误、用户不存在、账号锁定等场景

### 4.2 接口测试
- 使用 Postman 对登录接口进行测试
- 验证参数为空、超长、特殊字符等边界情况

### 4.3 自动化测试
- 使用 Selenium+Python 编写登录功能自动化测试脚本
- 实现账号输入、密码输入、点击登录、结果验证的全流程自动化

### 4.4 缺陷管理
- 使用 Jira 跟踪缺陷
- Bug 状态流转：新建 → 已分配 → 已修复 → 已关闭

## 五、测试入口与退出标准

### 入口标准
- 开发完成功能开发，代码已部署到测试环境
- 开发自测通过

### 退出标准
- 所有P0/P1缺陷已修复并验证通过
- 所有测试用例已执行完毕
- 测试报告已输出

## 六、风险评估

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 测试环境不稳定 | 测试中断 | 及时联系运维修复 |
| 开发修复不及时 | 测试延期 | 每日跟进修复进度 |

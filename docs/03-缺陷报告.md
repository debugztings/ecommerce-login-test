# 电商网站登录模块缺陷报告

## Bug 1：参数校验漏洞 - 用户名允许空字符串提交

| 字段 | 内容 |
|------|------|
| **标题** | 【登录页】用户名为空字符串时可提交登录请求 |
| **严重等级** | 严重 |
| **优先级** | 高 |
| **环境** | Chrome 120 / 测试环境 v1.2.0 |
| **状态** | 已关闭 |

### 复现步骤
1. 打开登录页面
2. 用户名输入框输入一个空格字符
3. 密码输入 Test123
4. 点击登录

### 预期结果
- 提示"用户名不能为空"

### 实际结果
- 请求正常提交到后端，返回"用户不存在"
- 问题：后端没有对空格字符进行trim处理，导致空格被当作有效用户名提交

### 发现时间
2024.03.05

## Bug 2：参数校验漏洞 - 密码超长未拦截

| 字段 | 内容 |
|------|------|
| **标题** | 【登录页】密码超过20位时后端未做长度校验 |
| **严重等级** | 严重 |
| **优先级** | 高 |
| **环境** | Chrome 120 / 测试环境 v1.2.0 |
| **状态** | 已关闭 |

### 复现步骤
1. 打开登录页面
2. 输入用户名 test001
3. 输入21位密码：123456789012345678901
4. 点击登录

### 预期结果
- 前端提示"密码长度不能超过20位"

- 前端未做长度限制，请求直接提交到后端
- 后端也未拦截，返回"用户不存在"
- 问题：前后端均未做密码长度上限校验

2024.03.05

1. 打开登录页面

## Bug 4：账号锁定功能异常

| 字段 | 内容 |
|------|------|
| **标题** | 【登录页】账号锁定次数阈值配置错误，3次即锁定 |
| **严重等级** | 一般 |
| **优先级** | 中 |
| **环境** | Chrome 120 / 测试环境 v1.2.0 |
| **状态** | 已关闭 |

### 复现步骤
1. 输入错误密码第1次 → 提示"密码错误"
2. 输入错误密码第2次 → 提示"密码错误"
3. 输入错误密码第3次 → 提示"账号已被锁定"

### 预期结果
- 应允许连续5次错误后才锁定

### 实际结果
- 连续3次错误即被锁定
- 问题：需求文档要求5次，但实际配置为3次

### 发现时间
2024.03.07

## Bug 5：记住密码功能异常

| 字段 | 内容 |
|------|------|
| **标题** | 【登录页】勾选"记住密码"后未自动填充用户名 |
| **严重等级** | 一般 |
| **优先级** | 中 |
| **环境** | Chrome 120 / 测试环境 v1.2.0 |
| **状态** | 已关闭 |

### 复现步骤
1. 登录页面勾选"记住密码"
2. 输入用户名密码，登录成功
3. 关闭浏览器
4. 重新打开登录页面

### 预期结果
- 用户名自动填充

### 实际结果
- 用户名和密码均未自动填充

| 字段 | 内容 |
|------|------|
| **标题** | 【登录页】登录接口平均响应时间超过3秒 |
| **严重等级** | 一般 |
| **优先级** | 中 |
| **环境** | Chrome 120 / 测试环境 v1.2.0 |
| **状态** | 已关闭 |

### 复现步骤
1. 使用JMeter对登录接口进行压力测试
2. 50并发，循环10次
3. 查看聚合报告

- 平均响应时间 < 500ms

### 实际结果
- 平均响应时间 2100ms
- 95%百分位 4500ms
